集群搭建

云计算结课已经有一段时间了，现在回想起来当时搭建集群的过程真是心有余悸。全程都是在报错和搜索中度过，现在想对这个痛苦的过程做一个简单的小总结，尤其是中间踩的坑。

不过，最最重要的一点是一定要在Linux环境下配置集群！

Hadoop集群搭建

前置配置

SSH配置

本机自身无密码登陆

  • 生成公钥、私钥对

    ssh-keygen -t rsa

  • 创建“authorized_keys”文件，并修改权限为“600”

    chmod 600 ~/.ssh/authorized_keys

  • 追加公钥到“authorized_keys”文件中

    cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

不同主机无密码登陆

  • 首先保证每台主机都生成了公钥、私钥对

  • 使用scp命令，将A主机公钥发送给B主机，将B主机公钥发送给A主机。

    scp ~/.ssh/id_rsa.pub username@ip:~/.

  • 追加公钥到“authorized_keys”文件中

    cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

  • 需要保证需要互相登陆的主机都拥有彼此的公钥

安装Java

网上关于Linux环境下安装Java的博文已经有很多了，这里就不赘述了，只说一下需要注意的点。

注意：集群中的所有机器的Java版本必须一样，安装路径必须相同

安装Hadoop

  1. 从 hadoop官网 下载当前最新的文件，Binary版本是二进制版本，直接解压安装； src 是 Hadoop
    源代码，需要编译才可安装。

    wget http://mirrors.nju.edu.cn/apache/hadoop/common/hadoop-3.2.1/hadoop-
3.2.1.tar.gz

  2. 将hadoop压缩包移动到合适的目录下并解压，这里选了/usr/local，后面的命令需要sudo权限

    sudo mv hadoop-3.2.1.tar.gz /usr/local
sudo tar -zxvf hadoop-3.2.1.tar.gz
sudo mv hadoop-3.2.1 hadoop
sudo chown -R hadoop:hadoop hadoop

  3. 将hadoop程序添加到环境变量

    echo 'export HADOOP_HOME=/usr/local/hadoop' >> /etc/profile
echo 'export HADOOP_CONF_DIR=$HADOOP_HOME/etc/hadoop' >> /etc/profile
echo 'export PATH=$PATH:/usr/local/hadoop/bin:/usr/local/hadoop/sbin' >>
/etc/profile
. /etc/profile
update-alternatives --display java
vim /usr/local/hadoop/etc/hadoop/hadoop-env.sh

    将54行修改为export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64

  4. 验证安装

    hadoop version

Hadoop集群配置

前置配置

修改host 文件，例如

vim /etc/hosts
192.168.43.147 hadoop-master
192.168.43.37 hadoop-slave1

注意：

  • 如果是Ubuntu系统，需要将 127.0.0.1 行注释掉，否则会在集群启动的时候会只监听本地端
    口，其他节点无法成功连接到它

  • 如果是Centos系统，不可以注释掉127.0.0.1 ,否则会在集群启动的时候报 datanode 端口监
    听错误，监听 0:0端口无效

主节点配置

  1. 配置core-site.xml，写入

    <configuration>
	<property>
	<!--用来指定使用hadoop时产生文件的存放目录-->
		<name>hadoop.tmp.dir</name>
		<value>file:/usr/local/hadoop/tmp</value>
		<description>Abase for other temporary directories.
		</description>
	</property>
	<property>
	<!--指定namenode的地址-->
		<name>fs.defaultFS</name>
		<value>hdfs://hadoop-master:9000</value>
	</property>
</configuration>

  2. 配置hdfs-site.xml，写入

    <configuration>
	<property>
		<name>dfs.replication</name>
		<value>2</value>
	</property>
	<property>
		<name>dfs.namenode.name.dir</name>
		<value>file:/usr/local/hadoop/tmp/dfs/name</value>
	</property>
	<property>
		<name>dfs.datanode.data.dir</name>
		<value>file:/usr/local/hadoop/tmp/dfs/data</value>
	</property>
</configuration>

  3. 配置mapred-site.xml，写入

    <configuration>
	<property>
		<name>mapreduce.framework.name</name>
		<value>yarn</value>
	</property>
	<property>
		<name>mapred.job.tracker</name>
		<value>http://hadoop-master:9001</value>
	</property>
	<!-- 如果没有以下3行，分布式运行的时候可能会报文件路径找不到等类似错误 -->
	<property>
		<name>yarn.app.mapreduce.am.env</name>
		<value>HADOOP_MAPRED_HOME=/usr/local/hadoop</value>
	</property>
	<property>
		<name>mapreduce.map.env</name>
		<value>HADOOP_MAPRED_HOME=/usr/local/hadoop</value>
	</property>
	<property>
		<name>mapreduce.reduce.env</name>
		<value>HADOOP_MAPRED_HOME=/usr/local/hadoop</value>
	</property>
</configuration>

  4. 配置yarn-site.xml，写入

    <configuration>
	<!-- Site specific YARN configuration properties -->
	<property>
		<name>yarn.nodemanager.aux-services</name>
		<value>mapreduce_shuffle</value>
	</property>
	<property>
		<name>yarn.resourcemanager.hostname</name>
		<value>hadoop-master</value>
	</property>
	<!--下列内容根据实际配置 -->
	<!-- 分配给hadoop的内存资源，注意该值应该小于(不建议等于)系统的总内存,或者使用`free
-h` 查看系统当前可用的系统资源，该值请不要小于1536 -->
	<property>
		<name>yarn.nodemanager.resource.memory-mb</name>
		<value>1600</value>
	</property>
	<!-- 分配给hadoop的CPU数量 -->
	<property>
		<name>yarn.nodemanager.resource.cpu-vcores</name>
		<value>1</value>
	</property>
	<!-- 以下内容炸鸡根据自身情况配置 -->
	<!-- hadoop的虚拟内存:上面分给hadoop的实际内存，默认2.1,当系统是1C2G时，该值不要
小于2.4 -->
	<property>
		<name>yarn.nodemanager.vmem-pmem-ratio</name>
		<value>3</value>
	</property>
	<!-- 每个任务的最小分配内存 -->
	<property>
		<name>yarn.scheduler.minimum-allocation-mb</name>
		<value>400</value>
	</property>
	<!-- 每个任务的最大分配内存，该值请不要小于1536 -->
	<property>
		<name>yarn.scheduler.maximum-allocation-mb</name>
		<value>1600</value>
	</property>
</configuration>

  5. 配置workers 文件，写入worker的主机名即可

配置从节点

将主节点的Hadoop文件夹拷贝给从节点主机即可

启动集群

  1. 格式化节点

    hdfs namenode -format

  2. 在任何一个节点执行start-all.sh

  3. 主节点的 http://masterip:9870 查看集群状态

以上为Hadoop节点的搭建内容，如果保证在Linux环境下，且节点的Hadoop和Java安装路径与安装版本都一样，应该是不会出现问题的。

Spark集群搭建

前置配置

Scala配置

同上文的Java配置，保证版本与路径相同

spark配置

  1. 在官网下载与前面下载的Scala版本对应的spark版本，版本一定要对应，不然会报错

  2. 在spark安装路径下，找到conf文件夹，复制spark-env.sh.template并重命名为spark-env.sh，配置``spark-env.sh`

    
export SCALA_HOME=/usr/scala //scala安装路径，自行更改
export SPARK_HOME=/usr/spark //spark安装路径，自行更改
export JAVA_HOME=/usr/java/jdk //Java安装路径，自行更改
 
export HADOOP_CONF_DIR=/usr/hadoop/etc/hadoop //Hadoop安装路径，自行更改
#export SPARK_EXECUTOR_INSTANCES=2
#export SPARK_EXECUTOR_CORES=2
#export SPARK_EXECUTOR_MEMORY=2G
export SPARK_WORKER_MEMORY=2G   
#export SPARK_YARN_APP_NAME=Spark
#export SPARK_YARN_QUEUE=default
export SPARK_MASTER_IP=192.168.0.100 //主机ip，自行更改

  3. 把slaves.template拷贝为slaves,并编辑 slaves文件，内容根据自己的集群节点名字更改

    hadoop-master
hadoop-slave1
hadoop-slave2

  4. 将上面的配置同步到所有节点

spark集群启动

在spark安装目录的sbin目录下执行

./start-all.sh
spark集群检查

访问http://masterip:8080/

总结

其实如果能注意到在Linux环境下并保证节点主机的路径与版本相同，配置起来并不是特别麻烦。当初最开始配置的时候，在Windows环境下折腾了半天，只配出来了一个伪分布式的集群，最后换成Linux虚拟机，半小时不到就搞定了。Linux天下第一！
